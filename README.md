# ToUniBus

✅ Cadastro e login de alunos.

✅ Envio de documentos em PDF ou imagem.

✅ Solicitação de vaga no transporte.

✅ Aprovação ou rejeição por um administrador da prefeitura.

✅ Consulta do status da solicitação.

✅ Relatório de alunos aprovados por rota.

✅ Limite de vagas por ônibus.

Tecnologias que fazem sentido
Backend: ASP.NET Core
ORM: Entity Framework Core
Banco de dados: PostgreSQL ou SQL Server
Documentação: Swagger/OpenAPI
Autenticação: JWT.
Algumas regras de negócio interessantes
O aluno não pode ter mais de uma solicitação ativa no mesmo semestre.
O número de aprovados não pode ultrapassar a capacidade do ônibus.
Apenas administradores podem aprovar solicitações.
O aluno deve enviar os documentos obrigatórios antes de solicitar uma vaga.

Esse projeto é forte para portfólio porque demonstra:

Modelagem de banco de dados;
Autenticação e autorização;
Upload de arquivos;
APIs REST;
Regras de negócio reais;
Geração de relatórios.

Se quiser deixar o projeto ainda mais profissional, adicione:

E-mails automáticos de aprovação/rejeição;
Histórico de renovações semestrais;
Lista de espera quando as vagas acabarem;
Logs de auditoria (quem aprovou ou rejeitou uma solicitação).
